Module 1

Deadlock Detection

Provide the current state of the system. The algorithm checks whether any subset of processes is deadlocked.

📖 Algorithm

Detection Algorithm — Pseudocode

Work  ← Available
Finish[i] ← (Allocation[i] == 0) for each i

repeat:
  find i such that Finish[i] = false AND Request[i] ≤ Work
  if found:
    Work   ← Work + Allocation[i]
    Finish[i] ← true
until no such i found

Deadlocked ← { i | Finish[i] = false }

Module 2

Deadlock Prevention — Banker's Algorithm

Enter Maximum and Allocation matrices. The system computes Need automatically and determines if the state is safe.

📖 Algorithm

Banker's Algorithm — Pseudocode

Need[i]  ← Maximum[i] − Allocation[i]
Work     ← Available
Finish[i]← false for each i

repeat:
  find i such that Finish[i] = false AND Need[i] ≤ Work
  if found:
    Work    ← Work + Allocation[i]
    Finish[i] ← true
until no such i found

Safe ← all Finish[i] = true

About This Project

📚 Academic Context

Created as an Operating Systems mini project to demonstrate deadlock concepts including detection and avoidance using classical algorithms taught in OS courses.

🛠 Tech Stack

Backend: Python · Flask
Frontend: HTML5 · CSS3 · Vanilla JS
Algorithms: Resource Allocation Graph · Banker's Algorithm

📐 Complexity

Detection: O(n² · r) time, O(n + r) space
Banker's: O(n² · r) time, O(n · r) space
where n = processes, r = resource types

🎓 References
